Downloading Phenote

For users

Phenote installer edition

Platform-specific installers are available for Phenote v1.7-b12 at SourceForge or here: MacOS X | Windows | Linux | Unix
  1. Download your platform-specific package. (For MacOS, this should self-extract into a disk image. If it doesn't, double-click on the file. You should see an icon for the "drive" that the Phenote application is on.)
  2. Double-click on the Phenote installer icon. This will open up the installer software. You will be asked a few questions, such as the installation location.
  3. When it's finished installing, you are ready to go! Double-click the Phenote application icon to launch Phenote.

We will continue to post links to major releases on this website. You can also install new releases from the Phenote SourceForge Project page.

Webstart edition

The webstart edition is currently unavailable while we resolve some problems. Please use the installer edition instead.

Note about Phenote and Phenote+

There's a small nomenclature change starting in v1.7. In 2008, the Phenote developers introduced a new "look and feel" to the Phenote user interface. They called this new version "Phenote+". At that time, Phenote+ was considered a still-experimental version, and "regular" Phenote was still available. We have decided that it's time to retire the old Phenote interface and make the Phenote+ interface the default. (Phenote+ will henceforth be referred to as "Phenote"; there won't be anything called "Phenote+".)

For developers

Check out our special documentation for running Phenote from the command line or in Eclipse

Hardware requirements

  • 400 Mhz processor or faster
  • 500+ MB RAM
  • 200 MB free disk space on your computer
  • Internet connection (broadband is best). Required for automatic updates of ontologies and some functions.

Software requirements

  • Any of these operating systems: Windows XP; Mac OSX; Linux; Solaris
  • Java 1.5 or later
  • To run from the command line: Ant
Recently added features and bug fixes, Phenote version 1.7-b11 and 1.7-b12
  • Can run even if you're offline
  • Will now warn you when you try to load a new data file if you've made changes in the current file.
  • Term info browser now syncs with other components (such as Graph Editor) to show the selected term when we follow parent/child hyperlinks in the Term Info window.
  • Now you can double-click a term in the annotation table and the Term Info browser AND any open OBO-Edit components such as the Graph Editor will change to show that term. Before, you had to edit the term and get a dropdown completion list and pick something from it (even if you just picked what was already there) before you could convince the Term Info and other windows to show the term.
  • (b12) Terms used in annotations that are not in any currently loaded ontology are no longer lost (and are now visually indicated in the Term Info window).
